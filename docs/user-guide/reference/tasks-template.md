# タスク分割テンプレート（6フェーズ版）

このドキュメントは、要件定義・設計・実装・試験・リリース準備・リリースの**6フェーズすべて**を含むtasks.mdのテンプレートです。

## 概要

**重要**: tasks.mdは以下の6フェーズを含める必要があります：

1. **Phase 0: 要件定義（Requirements）**
2. **Phase 1: 設計（Design）**
3. **Phase 2: 実装（Implementation）** - プロジェクトセットアップからコア機能実装まで
4. **Phase 3: 試験（Testing）**
5. **Phase 4: リリース準備（Release Preparation）**
6. **Phase 5: リリース（Release）**

### 完了条件について

**完了条件（Acceptance Criteria）**は、そのStoryが「完了」とみなされるために満たすべき条件です。

- ✅ **正しい理解**: 「そのJIRAタスクが完了するための条件」
- すべての完了条件を満たした時点で、Storyは「完了」とみなされます
- レビュー時には、完了条件を1つずつ確認して完了を判断します

## フェーズヘッダーの形式

```markdown
## Phase X: フェーズ名（ラベル）
```

**重要**: `（ラベル）`部分は必須です。JIRAストーリーのラベル検出に使用されます。

### 例

```markdown
## Phase 0: 要件定義（Requirements）
## Phase 1: 設計（Design）
## Phase 2: 実装（Implementation）
## Phase 3: 試験（Testing）
## Phase 4: リリース準備（Release Preparation）
## Phase 5: リリース（Release）
```

## JIRAラベル一覧

各ストーリーには以下のラベルが自動付与されます：

- `requirements` - 要件定義フェーズ
- `design` - 設計フェーズ
- `dev` - 実装フェーズ
- `testing` - 試験フェーズ
- `release-prep` - リリース準備フェーズ
- `release` - リリースフェーズ

**JIRAフィルター例**:
```text
project = PROJ AND labels = "requirements"  # 要件定義タスクのみ
project = PROJ AND labels = "testing"       # 試験タスクのみ
project = PROJ AND labels = "release"       # リリースタスクのみ
```

3. JIRAに全フェーズのストーリーが自動作成される
