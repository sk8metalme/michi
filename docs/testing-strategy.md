- **期間**: 2025年2月

### Phase 3: 統合テストの強化
- **目標カバレッジ**: 80%

- **目標カバレッジ**: 95%

## 現在の除外項目

以下のファイルはカバレッジ計測から除外されています（`vitest.config.ts`）:

- `node_modules/`
- `dist/`
- `**/*.test.ts`, `**/*.spec.ts`
- `**/__tests__/**`
- `**/*.d.ts`
- `**/*.config.ts`
- `**/scripts/setup-*.ts`
- `**/scripts/setup-*.sh`

## CI/CDでの扱い

### 現在のアプローチ
- ローカル開発: カバレッジ閾値30%で実行
- CI: 同じ閾値を適用し、段階的に引き上げ

### 推奨事項
- 新規コードには必ずテストを追加
- PRマージ時にカバレッジ低下を許可しない
- 月次でカバレッジ改善タスクを計画

## テスト実行コマンド

```bash
# すべてのテストを実行
npm run test

# テストを1回だけ実行（CI用）
npm run test:run

# カバレッジレポート付きで実行
npm run test:coverage

# UI付きでテストを実行（開発用）
npm run test:ui
```
